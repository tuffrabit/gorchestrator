Phase 4 Implementation Plan — Extensibility

📅 July 8, 2026 📝 Status: Draft 🎯 Prerequisite: Phase 3 complete

Goal

An issue filed in Jira flows into the daemon, the pipeline runs against a real git branch, the implementer iterates through container-sandboxed test-and-fix loops using an internal API exposed over MCP, and the resulting branch is pushed for human review.

Workstreams

A
Git Workspace Model (replaces §6.6 snapshots)
  • Project git config (spec §6.3): repo_url, base_branch, auth mode (ssh_key / token / gh_cli); environment prerequisites validated at project registration, not mid-run.
  • Clone cache per project; fetch before each run.
  • Branch per implementer run: ai-implementer/{issue_id}-{run_id}; workspace = checkout (replaces per-issue snapshot copies).
  • Post-run: stage, single structured commit (§17 Q13), optional push; human decides merge. Optional PR via gh.
  • Parallel implementers: branch + workspace per run, tracked in SQLite.
  • Symlink/containment hardening from cleanup assumed — real repos contain symlinks.
B
run_test — container sandbox is the acceptance criterion
Security posture (spec §6.5): the command executes code the implementer just wrote — arbitrary code execution by proxy. Container required: no network, workspace-only mount, CPU/memory/time limits. No bare-subprocess fallback — without a container runtime, run_test refuses to run.
  • Size-capped stdout/stderr returned to the agent; test-and-fix loop covered by an integration test.
  • Secrets injected from maintainer-only config into the container env; never rendered into task.json or any agent-readable artifact (§17 Q15).
C
MCP Client
  • MCP over stdio; servers declared in project config; deny by default.
  • Per-agent, per-server allowlists ship in this phase — required to land with external triggers, not after (spec §5.6). Per-tool granularity remains Phase 5.
  • Tools registered explicitly in agent config; MCP calls recorded in events.jsonl like core tools.
D
Trigger Port + External Triggers
  • Formalize TriggerPort (deferred from Phase 1); built-in webhook endpoint on the daemon.
  • External process adapters: GitHub Issues, Jira (JSON-RPC stdio, long-running listeners).
  • Untrusted input posture (spec §5.6): externally-triggered issues default to adjudicator: human on the implementation boundary; overridable per project, but the default is the safe one.
E
Storage & Agent Config
  • S3 external storage adapter — first real consumer of slash-canonical port keys and the events.jsonl write path.
  • Agent personality config (the casting thesis, realized): per-agent model/provider, temperature, prompt override, tool subsets — per-project defaults with per-issue overrides (settles §17 Q9).
  • Adapter authentication pattern (§17 Q12): adapters own their backend credentials via their own config/env — never through the core's agent-visible artifacts.
F
JSON-RPC Client Completion
  • Restart with exponential backoff on adapter death (deferred from cleanup).
  • Streaming notifications for long-running adapter events (trigger feeds).

Tests

  • End-to-end: webhook-submitted issue → pipeline on a fixture git repo → branch with commit exists; push against a local bare repo.
  • run_test: container enforced (refusal without runtime), timeout kill, no-network assertion, secrets absent from all artifacts.
  • MCP: fixture server; allowlisted agent sees tools, non-allowlisted does not; calls recorded in events.
  • Trigger adapter killed → restarted with backoff.
  • S3 adapter round-trip (e.g., MinIO) — keys identical across host platforms.

Success Criteria

  • Jira/GitHub issue → reviewed branch, hands-free except configured human gates.
  • Implementer completes a test-and-fix loop inside the sandbox.
  • An internal API is used by an agent via MCP under a per-agent server allowlist.
  • Externally-triggered issues stop at a human gate before implementation by default.

Out of Scope

  • Token budgets, effort estimation, scope detection → Phase 5
  • Per-tool MCP granularity → Phase 5
  • Azure Blob (follow-on once the S3 pattern is proven)
