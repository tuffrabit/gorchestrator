Phase 2 Project Cleanup — Defect Fixes & Spec Alignment

📅 July 8, 2026 📝 Status: Ready to Implement 🎯 Scope: No new pipeline features

Goal

Fix the defects found in the post-Part-1 code review and align the implementation with the revised spec (2026-07-08). After this phase the system still runs a single Researcher, but on the revised artifact contract with the known defects closed.

$ go run . run --issue="add auth" --project=foo --dry-run
→ Same behavior as today, except artifacts land in the revised layout
  (attempts/1/output.md, events.jsonl, result.json at start + completion)

Why a dedicated phase: Part 2 builds the phase machine directly on top of path containment, token accounting, the artifact layout, and the SQLite layer. Fixing these first means Part 2 is built on the final contract instead of churning it mid-build. Sequenced between Phase 2 Part 1 (✅ complete) and Phase 2 Part 2. Do not start Part 2 before this phase completes.

Group A — Security & Correctness Fixes

A1
Separator-aware path containment

internal/storage/fs.go uses strings.HasPrefix(realAbs, fs.root), so a root of /data/gorch admits /data/gorch-evil. Replace with a filepath.Rel-based check. Test: sibling directory whose name extends the root's is rejected.

A2
Symlink resolution policy

resolve() never calls EvalSymlinks; a symlink inside the root pointing outside defeats containment. Workspaces copied from real repositories will contain symlinks (Part 2), so this must close now. Resolve symlinks on the deepest existing ancestor, re-check containment, reject escapes. Tests: escaping symlinked file and escaping intermediate directory.

A3
Per-call token accounting

loopTokens = int(ev.UsageMetadata.TotalTokenCount) overwrites on each event — a multi-tool-call loop counts only the final model call. Accumulate per model call (trivial once events are recorded, see C2). Extend the dry-run model to a scripted multi-turn sequence and assert the total is the sum, not the last value.

A4
Truthful result.json
  • LoopCount reports the actual count reached, not the configured target, when a run fails mid-way.
  • Stop stuffing final output into done_rationale — reserved for self-adjudication (spec §7.2).
  • Ctrl-C produces status: cancelled, not failed (spec §7.3).
A5
Empty output fails the loop

Today, an agent that neither calls write_output nor produces final text yields a zero-byte output marked done. Per revised spec §13.1: that is a failed loop. Keep the fallback for non-empty final text.

A6
read_file: two modes + configurable cap

Restructure as two explicit modes (spec §12.1):

  • Whole-file: no range args → full file, subject to a cap. On truncation the result carries an explicit marker plus the file's total line count, so the agent knows to switch modes.
  • Surgical: offset (1-based line) + limit (line count) → exactly that range. The intended follow-up to grep_search, which already returns file + line numbers — grep to locate, then read the relevant region.

Cap is configurable (tools.read_file.max_bytes / max_lines) with sane defaults: 64KB / ~2,000 lines, whichever first. The tool's description string teaches the grep → surgical-read workflow to the model.

Group B — Robustness

B1
JSON-RPC client hardening
  • bufio.Scanner default 64KB line limit truncates large payloads — set a generous explicit buffer (~10MB); oversize is an error, not a silent skip.
  • Implement the initialize handshake from spec §4.3.
  • Close() blocks forever on a child ignoring stdin closure — wait with timeout, then kill.
  • Capture child stderr into core logs; adapter failures are currently invisible.
  • Deferred to Phase 4: restart with exponential backoff, streaming notifications.
B2
Explicit adapter registry

Per revised spec §4.3: adapters declared in config (name + manifest path); remove directory scanning. Manifest gains an explicit binary: field verified as a regular executable file — today os.Stat accepts a directory, and the repo's own adapters/noop.yaml next to the adapters/noop/ source directory passes discovery and would fail at exec. Remove the decorative Discovery() call from the orchestrator.

B3
OpenAI adapter resilience

Retry with exponential backoff on 429/5xx (respect Retry-After) — a single rate-limit blip currently fails an entire phase. Fall back to m.model when req.Model is empty. Streaming remains deferred.

B4
SQLite pragmas + versioned migrations

Open with journal_mode=WAL, busy_timeout (5s), foreign_keys=ON (silently unenforced without it). Replace accreting CREATE TABLE IF NOT EXISTS with a minimal versioned migration mechanism — Part 2's decisions table and Phase 3's tables arrive as migrations instead of schema drift.

Group C — Artifact Contract Alignment (spec §7)

C1
result.json at phase start

Write {"status": "in_progress"} before the agent runs; finalize on completion. This is the substrate for crash detection (spec §9.4) — recovery behavior itself lands in Part 2.

C2
events.jsonl transcripts

Append-only, one JSON object per line: model turns, tool calls and results (size-capped), per-call token usage, timestamps. Sourced from the ADK event stream the orchestrator already drains and currently discards. Written through the StoragePort so external storage adapters inherit it.

C3
attempts/ layout + feed-forward loops

Outputs move to attempts/1/output.md; result.json gains attempt and latest_output fields. Loop i receives loop i−1's output in context (spec §8.3); intermediate loop states are visible in events.jsonl. The single-researcher flow always uses attempt 1 — multiple attempts arrive with adjudication in Part 2, but the layout must not churn then.

C4
Slash-canonical storage keys

path.Join (forward slash) for port keys, not filepath.Join; the FS adapter translates to OS separators. Prevents separator leakage into future S3/Azure keys (spec §3.3).

C5
.gitignore-aware grep

Honor .gitignore as the Phase 1 plan promised (in addition to existing .git and binary skips). Matters as soon as source snapshots arrive in Part 2 (node_modules, build dirs).

Group D — Hygiene

D1
Dead code removal

Remove finishTaskResult, MarshalTask, lastOutput misuse, _ = loopStart. Consolidate the duplicated schemaToMap (orchestrator + llm packages) into one shared helper. mustTool panics — propagate errors to the CLI instead.

D2
README.md

What the project is, current status, how to run the dry-run demo, and the documentation convention (spec.md is living; phase docs freeze; markdown canonical).
