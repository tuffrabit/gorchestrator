gorchestrator

Phase 2 Implementation Plan — The Pipeline

📅 July 8, 2026 📝 Status: Split: Part 1 This Session 🎯 Scope: Research → Plan → Implement with configurable handoffs

Goal

Execute the full agent pipeline from the CLI. Each phase feeds its output.md into the next phase. Handoff boundaries are configurable per agent.

Trigger → Researcher → [Adjudicate] → Planner → [Adjudicate] → Implementer → Done
$ gorchestrator run --issue="add auth" --project=foo
→ Creates issue, runs researcher → planner → implementer, writes artifacts, records state in SQLite
Session Plan: Part 1 = ADK migration (this session). Part 2 = multi-agent pipeline + handoffs (follow-up session).

Locked-In Decisions

Topic Decision
Core Agent runtime Tight integration with google.golang.org/adk/v2. No wrapper/port abstraction.
Core LLM providers ADK's model.LLM interface. Gemini built-in via model/gemini; OpenAI via a custom model.LLM adapter. Old internal/llm port is removed.
Decision Session/state ADK session is ephemeral per phase. Filesystem is the only persisted memory/state. Use session.InMemoryService().
Decision Tools Native ADK function tools (tool/functiontool) with typed arg/result structs. Kept generic enough for core tools + future MCP tools.
Decision Implementer workspace Scratch directory under projects/{pid}/issues/{iid}/implementation/workspace/ for Phase 2. Git workspace model remains Phase 4.
Decision Human gate (CLI-only) Orchestrator writes waiting_human, exits, and records a pending decision. A resume subcommand applies pass/fail/retry.
Decision Phase split Part 1 = ADK migration. Part 2 = pipeline, agents, handoffs, adjudicators.

Part 1: ADK Migration (This Session)

1
Bootstrap ADK dependency

Fetch google.golang.org/adk/v2@v2.0.0 and verify the module still builds. Update example config to support Gemini/OpenAI provider selection.

2
Redesign tools as ADK function tools

Replace the custom tools.Tool interface with ADK's tool.Tool. Each tool is a functiontool.New[TArgs, TResult] wrapper around a typed Go function. Migrate read_file, list_directory, grep_search, and write_output. Preserve path allowlist enforcement inside the handlers.

3
Redesign LLM layer around ADK model.LLM

Remove the old provider port. Add:

  • internal/llm/gemini.go — build Gemini model from config.
  • internal/llm/openai.go — custom model.LLM that calls OpenAI chat completions and maps to/from genai types.
  • internal/llm/dryrun.go — canned model.LLM for tests.
  • internal/llm/factory.go — build a model from provider name.
4
Rebuild Researcher as an ADK LLMAgent

Remove the custom agent loop. NewResearcher(model model.LLM) returns an llmagent.New(...) configured in task mode with the researcher toolset and system prompt. The orchestrator drives it via ADK's runner and drains events to capture output and usage.

5
Rewrite orchestrator run flow for ADK

Keep project/issue/SQLite logic. For a single Researcher run, construct an ADK Runner with an in-memory session service, run the agent, extract the final output, write output.md and result.json, and record the run. Preserve --dry-run.

6
Update tests

Ensure the existing dry-run integration test passes with the ADK backend. Add a unit test for the OpenAI-to-genai translation layer.

Part 1 Success Criteria

  • go run . run --issue="add auth" --project=foo --dry-run completes and writes the same Phase 1 artifacts.
  • All existing tests pass.
  • go mod tidy produces a clean module.
  • No custom agent loop or old LLMProviderPort code remains.

Part 2: Multi-Agent Pipeline + Handoffs (Follow-Up Session)

7
Add Planner and Implementer agents

Build ADK LLMAgents for planner (researcher-style tools + write_output) and implementer (workspace write tools). Each has a system prompt tuned for the next consumer.

8
Add implementer tools

Implement write_file and update_file as ADK function tools. Paths are workspace-relative and constrained to implementation/workspace/.

9
Define Adjudicator port

Create internal/adjudication with an Adjudicator interface returning Pass/Fail/Retry. Implement NullAdjudicator and SelfAdjudicator.

10
Implement handoff modes

Support n_loops, self_done, and human_gate. self_done uses the agent's finish_task rationale. human_gate writes waiting_human and exits.

11
Build the phase machine

Orchestrator runs research → plan → implementation. Each phase reads the previous output.md, writes its own task.json, runs the ADK agent, writes output.md + result.json, then applies the configured adjudicator before proceeding.

12
SQLite decisions table + resume command

Add a decisions table. Implement gorchestrator resume --project=foo --issue=N --decision=pass|fail|retry to continue a waiting_human issue.

13
External adapter examples

Add a webhook trigger adapter and an email notification adapter under adapters/. They exercise the existing JSON-RPC plumbing but are not wired into the core flow in Phase 2.

14
Tests

Add end-to-end dry-run pipeline test, adjudicator unit tests, resume flow test, and implementer path-enforcement tests.
