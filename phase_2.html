Goal

Execute the full agent pipeline from the CLI against real project source. Each phase feeds its accepted output into the next; every boundary uses the unified adjudication model with feedback-carrying retries.

Trigger → Researcher → [Adjudicate] → Planner → [Adjudicate] → Implementer → Done
$ gorchestrator run --issue="add auth" --project=foo --source=/path/to/repo
→ Snapshots source, runs researcher → planner → implementer against it,
  adjudicates each boundary, writes artifacts, records state in SQLite

Part 2: Multi-Agent Pipeline + Adjudication

Prerequisite: Phase 2 Project Cleanup complete — Part 2 assumes the attempts/ layout, events.jsonl, in-progress result.json, feed-forward loops, and hardened path containment are already in place.

7
Project source snapshot (spec §6.6)

Project config gains source_path. On issue creation the orchestrator copies it to source/ (excluding .git), adds it to every agent's read allowlist, and seeds the implementer's workspace/ from it — implementation edits real code, not an empty directory.

8
Planner and Implementer agents

ADK LLMAgents: planner with researcher-style tools + write_output; implementer with read_file, list_directory, grep_search, write_file, update_file. Prompts tuned for the next consumer. All agents move to ModeTask; the finish_task rationale feeds self-adjudication.

9
Wire implementer tools

write_file/update_file exist from cleanup; wire NewImplementerRegistry with the workspace path. Paths constrained to implementation/workspace/ under the hardened containment.

10
Adjudicator port (unified model)

Evaluate(ctx, phase, attempt, task) → Decision{Outcome, Feedback}. Null (auto-pass after loops), Self (reads finish_task rationale vs. rubric). Per-boundary config: adjudicator + max_attempts + loops — no separate handoff-mode setting.

11
Phase machine
  • Recovery scan first (spec §9.4): in-progress phases are treated as crashed and re-run.
  • Context per spec §8.3: issue + previous accepted output; loops feed forward.
  • Per phase: in-progress result.json → task.json → run agent → attempt output + events.jsonl → adjudicate.
  • Retry: feedback.md written, attempt N+1 gets rejected output + feedback in context.
  • Pass → next phase · Fail/exhausted → failed · human boundary → waiting_human · Ctrl-C → cancelled.
12
Decisions table + resume command

decisions migration includes a feedback column. resume --decision=pass|fail|retry [--feedback="..."]: pass finalizes and continues; retry starts a new attempt with feedback injected (warn when omitted); fail stops the issue. Pipeline position reconstructed from filesystem + SQLite (filesystem authoritative).

13
Anthropic model.LLM implementation

internal/llm/anthropic.go: Anthropic Messages API ↔ genai types, mirroring the OpenAI adapter including retry/backoff. Registered as provider anthropic.

14
Tests
  • End-to-end dry-run pipeline: attempts layout, context chaining, workspace seeding.
  • Adjudicator units (null, self, retry-with-feedback); resume flows (pass/retry/fail).
  • Path enforcement incl. symlink escape from a snapshot-seeded workspace.
  • Crash recovery: kill mid-phase → restart → phase re-runs → pipeline completes.
  • Anthropic translation unit test (no network).

Artifact Contract (post-cleanup layout)

{storage_root}/projects/{project_id}/issues/{issue_id}/
├── source/ ← read-only snapshot (orchestrator, at issue creation)
├── research/
│ ├── task.json ← orchestrator writes
│ ├── result.json ← in_progress at start; final on completion
│ ├── events.jsonl ← model turns, tool calls, per-call usage
│ └── attempts/1/
│ ├── output.md ← agent writes
│ └── feedback.md ← only if rejected
├── plan/ ← same structure
└── implementation/
    ├── (same structure)
    └── workspace/ ← seeded from source/; implementer writes here

Success Criteria

  • Full pipeline dry-run against a snapshot writes all artifacts in the revised layout.
  • Implementer workspace starts as a snapshot copy and contains its edits afterward.
  • resume --decision=retry --feedback="missing tests" re-runs with feedback in context.
  • Kill-mid-phase → restart → recovery (automated test).
  • go test ./... passes; no custom agent loop, old port, or ModeChat agents remain.
  • Per-model-call token usage and run history recorded per phase.

Out of Scope

  • Git workspace model → Phase 4 (snapshot copies are the interim)
  • run_test → Phase 4 (container sandbox is an acceptance criterion there)
  • MCP → Phase 4 · Dashboard/auth/daemon → Phase 3 · Budgets → Phase 5
  • Notification delivery → Phases 3/4; webhook/email example adapters cut from this phase
